# 概要
開発プロセスの成果物（要件、設計、コード、テストなど）の間に、前後関係や主従関係といった「関係性」を確立し、それを追跡できる度合い
## 要求事項のトレーサビリティ（Requirements Traceability）
一つの要件がその発生源（顧客のニーズやビジネス目標）から、設計、実装（コード）、そして最終的なテスト・検証に至るまでの全ライフサイクルを追跡できる能力を指します。
# 手法
**方向による分類**:
- **トレースフォワード**: 要件から設計・コード・テストへと、開発の進捗に沿って追跡します。これにより「全ての要件が漏れなく実装・テストされているか」を確認できます。
- **トレースバック（リバーストレース）**: テスト結果や特定の機能から、元となる要件へと遡ります。これにより「実装された機能がどの要件に基づいているか」や「余計なコードが含まれていないか」を確認できます。
- **双方向トレーサビリティ**: 上記の両方を可能にする状態です。複雑なシステムや安全性が重視される開発では、この双方向性が強く求められます。
**階層による分類**:
- **垂直トレーサビリティ**: 抽象度の高い「ビジネス要件」から、詳細な「技術仕様」「設計」「ソースコード」「テストケース」といった異なる階層を縦に繋ぐ追跡です。
- **水平トレーサビリティ**: 同じ開発フェーズ内（例：要件間、あるいはモジュール間）での依存関係や相互作用を横に繋いで管理することです。
# 役割
- **変更管理（Change Management）**: 仕様変更が発生した際、影響を受ける設計・コード・テスト範囲を即座に特定する「影響範囲分析（インパクト解析）」を可能にします。これにより、修正漏れや予期せぬ不具合（デグレード）を防ぎ、手戻りコストを最小限に抑えます。
- **品質・リスク管理**: 顧客の要求が漏れなく実装され、かつ適切にテストされたかを「網羅性（カバレッジ）」の観点から客観的に評価します。不備の早期発見や、安全性が重視されるシステムにおける品質保証の強力な証跡（エビデンス）となります。
- **進捗・工程管理**: 開発プロセス全体の進捗を可視化します。どの要求が実装済みで、どのテストが未完了かをリアルタイムで把握できるため、プロジェクトの現状をステークホルダーへ正確に報告し、意思決定を迅速化するツールとして機能します。
