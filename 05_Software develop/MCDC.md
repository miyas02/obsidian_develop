# 概要
判定網羅、条件網羅では不十分
複合条件網羅(Multiple Condition Coverage: MCC)は理論上は堅牢だが、条件の増加に伴って、テストケースが指数関数的に増加する。
MC/DCは、各条件が判定結果に「独立して影響を与える」ことを証明することを要求する。
複合条件網羅に近い検証強度を維持しつつ、テストケース数を条件数 $n$ に対して線形（最小 $n+1$ 個）に抑えることを可能にした。
# 構造的網羅率の比較と特性

| **網羅率の名称**            | **定義**                  | **検出可能な不備**           | **限界と課題**                         |
| --------------------- | ----------------------- | --------------------- | --------------------------------- |
| **命令網羅 (SC)**         | 全ての実行文を少なくとも1回実行する。     | デッドコード、到達不能な命令文。      | if文のelse句がない場合の論理欠落を検出できない。       |
| **判定網羅 (DC)**         | 各判定（分岐）の真と偽の結果を網羅する。    | 分岐の未実行、制御フローの誤り。      | 判定内の個別の条件が結果に与える影響を無視する。          |
| **条件網羅 (CC)**         | 判定内の各条件が真と偽の両方を取る。      | 条件式の誤り、単一条件の境界値不備。    | 判定全体の結果が真偽の両方になるとは限らない。           |
| **条件/判定網羅 (C/DC)**    | 各条件が真偽を取り、かつ判定全体も真偽を取る。 | 条件と判定の両方の基本的な実行。      | 論理演算子の誤り（ANDをORと記述など）を見逃す可能性がある。  |
| **改良条件/判定網羅 (MC/DC)** | 各条件が判定結果に独立して影響することを示す。 | 複雑な論理構造内の条件の誤り、冗長な条件。 | 論理構造の理解と高度なテスト生成が必要。              |
| **複合条件網羅 (MCC)**      | 全ての条件の組み合わせを網羅する。       | 論理式の完全な真理値表の検証。       | 条件数 $n$ に対し $2^n$ のテストを要し、実用的でない。 |

# MC/DCの論理的定義と四つの必須要件
1. **全入口/出口の実行**：プログラムの全てのサブルーチンやモジュールの入口点と出口点が少なくとも一回は呼び出されていること。
2. **判定の網羅**：判定式（例：if文の全体）が、可能な全ての出力結果（真および偽）を少なくとも一回は取ること。
3. **条件の網羅**：判定内に含まれる全ての個別の条件が、真および偽の値を少なくとも一回は取ること。
4. **独立した影響（Independent Effect）**：判定内の各条件が、判定の出力に独立して影響することを示すこと。   
**独立して影響する**：とは、「他の条件の値を適切に固定し、ある特定の条件の値を反転させたとき、それに連動して判定全体の結果も反転すること」を指す
# 独立性の証明アプローチ：Unique CauseとMasking
## Unique Cause MC/DC
対象となる条件のみを変化させることで判定結果が変化することを示す
`Z = (A AND B) OR C`
条件：Aの独立性を証明するには以下のペアが必要
case1:`A = true, B = true, C = false`
case2:`A = false, B = true, C = false
### Unique Causeの課題
Unique Causeは条件が結合(Coupled)している場合に網羅率を達成できないという致命的な欠点がある
**条件の結合とは、同一の変数が一つの判定式に複数回現れる**状態を指す。
`(A AND B) OR (A AND C)`
Aの値を固定したまま他方で変化させることが不可能。
## Masking MC/DC
対象外の条件が変化することを許容しつつ、その変化が判定結果に影響を与えない（マスキングされている）ことを確認する手法である。
マスキングの基本原理は、以下の支配法則（Domination Laws）に基づいている。
- AND演算：一つのオペランドが False であれば、他のオペランドの値に関わらず結果は False となる。
- OR演算：一つのオペランドが True であれば、他のオペランドの値に関わらず結果は True となる。
Masking MC/DCでは、独立性ペアの構成において、対象外の条件が変化したとしても、それらが論理的に判定結果を左右できない状況（マスキングされた状態）にあれば、独立性が証明されたとみなす
これにより、結合条件を持つ複雑な論理式であっても、一貫性のある検証が可能となる。
## Masking MC/DC 活用例
`(A and B) or C`
`(A or B) and C`
みたいなほかの演算の結果に影響をうける値：`C`の検証手法
`(A and B) や (A or B)などの演算結果が固定されていたらOK`
### `(A or B) and C`
Cの独立性の証明に：`A or B`の結果がtrueであれば、Cの結果を反転させればOK。
### `(A and B) or C`
Cの独立性の証明に：`A and B の結果がfalseであれば、Cの結果を反転させればOK。
